## 1. 문제 정의

### 현재 상황

동네 정보가 완전히 파편화되어 있습니다.

| 상황 | 불편함 |
|------|--------|
| 공사·교통 통제 | 구청 홈페이지를 직접 들어가 확인해야 함 |
| 사건·사고 | 뉴스 + 커뮤니티 이중 확인 필요 |
| 주말 행사 | 각 기관 SNS를 개별 팔로우해야 함 |
| 지역 뉴스 | 당근·네이버카페는 주민이 직접 써야 생기는 구조 |
| **소상공인 홍보** | 동네 가게 오픈·이벤트·할인 정보가 주민에게 닿지 못함 |
| **기업·브랜드 홍보** | 지역 타겟 광고를 집행하려 해도 하이퍼로컬 채널 자체가 없음 |

### 핵심 질문

> **"왜 날씨 앱처럼 매일 아침 동네 브리핑을 자동으로 받아볼 수 없는가?"**

## 2. 솔루션 개요

### 서비스명
- **dongne.me** (도메인 확정)
- 브랜드명: **동네** / **dongne**

### 한 줄 정의
> 공공 OpenAPI와 지역 뉴스를 자동 수집하고, AI가 동네별로 요약·정리해 매일 아침 브리핑 형태로 제공하는 하이퍼로컬 구독 서비스

### 핵심 가치 제안

```
수집 → 요약 → 브리핑
(자동)  (AI)   (매일 오전 8시)
```

- 사용자는 **아무것도 하지 않아도** 매일 아침 동네 소식 브리핑을 받는다
- 정보 출처가 공공기관이라 **신뢰도가 높다**
- **카테고리별 분류**로 관심 있는 정보만 볼 수 있다
- **소상공인·기업 홍보** 채널로도 활용 가능하다
- 향후 **관심사·생활패턴 기반 개인화 브리핑**으로 진화한다

## 3. 시장 조사 — 경쟁 포지셔닝

### 경쟁 지도

```
                    높은 자동화 (AI)
                         │
              [dongne.me]│[Patch.com]
                         │
공공·공식 ───────────────┼─────────────── UGC·주민
데이터 중심              │               콘텐츠 중심
                [빅카인즈]│[Nextdoor]
                         │    [당근]
                    낮은 자동화 (수동)
```

**→ 좌상단 포지션(공공 데이터 + 높은 자동화)은 국내에 아직 아무도 없다**

### 주요 유사 서비스

#### Patch.com — PatchAM (미국) — 가장 유사
- 우편번호 기반 AI 뉴스레터 자동 발송
- 30,000개 커뮤니티, 구독자 100만+, 인간 에디터 0명
- **시사점**: 동일한 모델이 미국에서 이미 검증됨

#### 당근마켓
- MAU 2,000만+이지만 UGC 구조 → 공공 정보 통합 없음
- **결론**: 경쟁이 아닌 보완 관계 가능

### 핵심 차별점

> **"만들어지길 기다리지 않고, AI로 자동화해서 가치를 직접 배달하는 동네 신문"**

## 4. 핵심 기능 명세

### 4-1. 8가지 브리핑 카테고리

| 카테고리 | 내용 | 데이터 소스 |
|----------|------|------------|
| 🌤 날씨 | 기온·강수·미세먼지 | 기상청 동네예보 API |
| 🚨 안전·치안 | 경찰청 신고 통계·화재 출동 | 경찰청·소방청 API |
| 🚧 교통·도로 | 공사 예고·교통 통제 | 구청 공공데이터 |
| 📢 행정·공지 | 구청·동사무소 공지 | 공공데이터포털 |
| 🎉 행사·이벤트 | 문화행사·주민센터 프로그램 | 문화체육관광부 API |
| 📰 지역 뉴스 | 지역 언론 + 커뮤니티 | 네이버 뉴스 API |
| 🏪 소상공인·기업 홍보 | 동네 가게 오픈·이벤트·할인 | 인허가 API + 광고 채널 |
| 👤 개인화 브리핑 *(예정)* | 관심사·생활패턴 맞춤 소식 | 사용자 설정 기반 |

### 4-2. 발송 채널

- **이메일**: 이메일 주소만으로 구독 (회원가입 없음, double opt-in)
- **KakaoTalk 친구톡**: 채널 친구 추가 → 매일 오전 8시 자동 발송 (앱 설치 불필요)
- **KakaoTalk 챗봇**: "오늘 브리핑", "구독 해지", "도움말" 명령어 지원

### 4-3. 대시보드 & 아카이브

- **경기도 시/군/구 필터 대시보드**: 지역별 브리핑 열람
- 날짜별 브리핑 아카이브 (비로그인 접근 가능)
- 지도 기반 UI *(Phase 2 예정)*
- 트렌드 분석 대시보드 *(Phase 3 예정)*

## 11. 소버린 AI 전략

dongne.me는 구조적으로 **소버린 AI(Sovereign AI) 적용에 최적화**된 아키텍처입니다.

### 왜 소버린 AI인가?

- **데이터 주권**: 국가 공공 인프라 데이터(기상청·경찰청·지자체)를 국내에서 수집·처리
- **문화·언어 맥락**: 한국어·지역 특성에 맞춘 AI 요약으로 글로벌 LLM의 문화적 편향 극복
- **보안 규정 준수**: 공공 분야 B2B 확장 시 국가 보안 규정 준수 용이

### 국내 소버린 AI 트렌드

KT·네이버·SK텔레콤 등 국내 기업들이 한국어 특화 LLM을 개발해 공공 분야에 적용 중. dongne.me는 이 생태계와 자연스럽게 연계 가능.

### 적용 방향

- 현재: Google Gemini API 사용 (비용 효율)
- Phase 3: 국내 LLM (하이퍼클로바X, Exaone 등) 검토 — 공공 B2B 계약 요건 충족용
